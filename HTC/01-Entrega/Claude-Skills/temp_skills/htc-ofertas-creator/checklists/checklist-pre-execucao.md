# Checklist Pré-Execução: Antes de Lançar sua Oferta High Ticket

Use este checklist antes de lançar qualquer oferta High Ticket para garantir que está pronta para converter.

---

## SEÇÃO 1: OFERTA (CORE)

### Transformação
- [ ] Transformação é específica (não genérica)
- [ ] Transformação é mensurável (tem números)
- [ ] Transformação tem timeframe definido
- [ ] Promessa é crível (não absurda)
- [ ] Resultado é desejado pelo público

### Mecanismo
- [ ] Metodologia tem nome único
- [ ] Metodologia é proprietária (não genérica)
- [ ] Processo é passo a passo (não conceitual)
- [ ] Diferenciação é clara vs concorrentes
- [ ] IP é defensável

### Componentes
- [ ] Mínimo 3 componentes principais
- [ ] Cada componente tem benefício claro
- [ ] Componentes são complementares (não repetitivos)
- [ ] Stack está completo (não faltam peças)
- [ ] Entregáveis são tangíveis (não vagos)

---

## SEÇÃO 2: VALOR E PREÇO

### Stack de Valor
- [ ] Valor total foi calculado (soma de tudo)
- [ ] Valor total é 2-3x o preço mínimo
- [ ] Cada componente tem valor individual
- [ ] Valores são realistas (baseados em mercado)
- [ ] Stack está claramente apresentado

### Estrutura de Preços
- [ ] Preço definido com justificativa
- [ ] 3 opções de pagamento disponíveis
  - [ ] À vista com desconto (10-15%)
  - [ ] Parcelamento curto (3-6x)
  - [ ] Parcelamento longo (10-12x)
- [ ] Preço termina em 7 (R$X.997)
- [ ] Preço é adequado ao nível (iniciante/intermediário/avançado)

### ROI Justificável
- [ ] Cliente tem ROI claro de 10x+
- [ ] ROI está explicado na oferta
- [ ] Comparação com alternativas mostra valor
- [ ] Custo de inação está destacado

---

## SEÇÃO 3: PROVA SOCIAL

### Depoimentos
- [ ] Mínimo 3 depoimentos com números
- [ ] Pelo menos 1 depoimento em vídeo
- [ ] Depoimentos têm antes/depois
- [ ] Depoimentos têm timeframe
- [ ] Depoimentos são recentes (últimos 12 meses)

### Casos de Sucesso
- [ ] Mínimo 1 caso estudo completo
- [ ] Caso tem números específicos
- [ ] Caso mostra journey completa
- [ ] Caso é crível e verificável

### Autoridade
- [ ] Seus credentials estão destacados
- [ ] Número de clientes servidos está visível
- [ ] Anos de experiência estão mencionados
- [ ] Resultados próprios estão destacados (se aplicável)

---

## SEÇÃO 4: GARANTIA

### Tipo de Garantia
- [ ] Tipo de garantia definido (incondicional/condicional)
- [ ] Prazo da garantia definido (30/60/90 dias)
- [ ] Termos da garantia estão claros
- [ ] Garantia protege cliente (remove risco)
- [ ] Garantia protege você (condições)

### Comunicação da Garantia
- [ ] Garantia está em seção destacada
- [ ] Termos estão em linguagem simples
- [ ] Processo de reembolso está claro
- [ ] Garantia está mencionada múltiplas vezes

### Credibilidade da Garantia
- [ ] Você consegue honrar se necessário
- [ ] Garantia baseada em confiança (não desespero)
- [ ] Raros usos da garantia esperados (<5%)

---

## SEÇÃO 5: ESCASSEZ E URGÊNCIA

### Escassez
- [ ] Razão genuína de limitação existe
- [ ] Número exato de vagas/slots está visível
- [ ] Vagas restantes estão destacadas
- [ ] Explicação do "por que limitado" está clara

### Urgência
- [ ] Deadline específico está definido
- [ ] Consequência de esperar está destacada
- [ ] Preço/offer muda após deadline (verdade)
- [ ] Urgência é crível (não falsa)

### Autenticidade
- [ ] Escassez é 100% genuína
- [ ] Urgência é 100% verdadeira
- [ ] Você pode provar se questionado
- [ ] Nada é "marketing inventado"

---

## SEÇÃO 6: COPY E MENSAGEM

### Headline
- [ ] Promessa principal está clara
- [ ] Headline tem especificidade
- [ ] Headline tem curiosidade
- [ ] Headline promete resultado
- [ ] Headline é única (não genérica)

### Estrutura
- [ ] Problema está agitado adequadamente
- [ ] Solução é apresentada como única
- [ ] Como funciona está claro (passo a passo)
- [ ] O que recebe está detalhado
- [ ] Prova social está distribuída

### Objeções
- [ ] Objeção "é caro" está endereçada
- [ ] Objeção "não tenho tempo" está endereçada
- [ ] Objeção "será que funciona?" está endereçada
- [ ] Objeção "posso fazer sozinho" está endereçada
- [ ] FAQ está completo (5-10 perguntas)

### Call to Action
- [ ] CTA é claro e específico
- [ ] CTA tem apenas 1 ação
- [ ] CTA está repetido múltiplas vezes
- [ ] CTA cria urgência
- [ ] CTA é fácil de executar

---

## SEÇÃO 7: ENTREGA E OPERAÇÃO

### Capacidade de Entrega
- [ ] Você tem capacidade para as vagas oferecidas
- [ ] Conteúdo está pronto/gravado
- [ ] Sistema de entrega está configurado
- [ ] Equipe (se houver) está treinada
- [ ] Timeline de entrega está clara

### Suporte
- [ ] Canais de suporte estão definidos
- [ ] Tempo de resposta está comunicado
- [ ] Equipe de suporte está pronta
- [ ] Ferramentas de suporte configuradas

### Qualidade
- [ ] Checklists de qualidade existem
- [ ] Processo de feedback está definido
- [ ] Métricas de sucesso estão claras
- [ ] Sistema de melhoria contínua existe

---

## SEÇÃO 8: TECNOLOGIA E SISTEMAS

### Infraestrutura
- [ ] Página de vendas está funcional
- [ ] Checkout/payment processa corretamente
- [ ] Emails de confirmação funcionam
- [ ] Área de membros está acessível
- [ ] Integrations estão testadas

### Tracking
- [ ] Analytics está instalado
- [ ] Pixels de ads configurados
- [ ] Conversões estão rastreando
- [ ] Dashboard de métricas existe
- [ ] Testes de transação foram feitos

### Automação
- [ ] Sequências de email configuradas
- [ ] Automações de follow-up ativas
- [ ] Segmentações estão criadas
- [ ] Reminders automáticos funcionando
- [ ] Webhooks/integrações testadas

---

## SEÇÃO 9: VENDA E CONVERSÃO

### Processo de Venda
- [ ] Script de venda está preparado
- [ ] Objection handling está documentado
- [ ] Equipe de vendas está treinada
- [ ] Processo de follow-up existe
- [ ] Sistema de agendamento funciona

### Qualificação
- [ ] Critérios de lead qualificado definidos
- [ ] Processo de qualificação existe
- [ ] Perguntas de descoberta preparadas
- [ ] Sistema de score de lead existe
- [ ] Disqualificação é aplicada

### Métricas
- [ ] Baseline de conversão está estabelecido
- [ ] Metas de conversão estão definidas
- [ ] Sistema de tracking de conversão existe
- [ ] Dashboard de vendas está ativo
- [ ] Review diário/semanal agendado

---

## SEÇÃO 10: LEGAL E COMPLIANCE

### Documentação
- [ ] Termos de uso estão completos
- [ ] Política de privacidade existe
- [ ] Política de reembolso está clara
- [ ] Contratos/prestação de serviço prontos
- [ ] Disclaimers apropriados estão visíveis

### Compliance
- [ ] Promessas são compliant (não exageradas)
- [ ] Testemunhais são autênticos (não inventados)
- [ ] Garantias são legais e executáveis
- [ ] Preços e condições são transparentes
- [ ] Práticas anti-spam são seguidas

---

## SCORE FINAL

Conte quantos items foram marcados:

- **90-100 marcados:** PRONTO PARA LANÇAR
- **70-89 marcados:** QUASE LÁ - revise o que falta
- **50-69 marcados:** PRECISA MELHORIAS SIGNIFICATIVAS
- **<50 marcados:** NÃO ESTÁ PRONTO - revise completamente

---

## AVISO FINAL

Se após completar este checklist você ainda tem dúvidas ou itens não marcados, considere:

1. Obter feedback de 3-5 pessoas do seu público-alvo
2. Fazer um teste beta com 5-10 pessoas
3. Consultar mentor/consultor especializado
4. Adiar lançamento até estar mais preparado

É melhor atrasar o lançamento e lançar algo ótimo do que lançar algo medíocre e arruinar reputação.
