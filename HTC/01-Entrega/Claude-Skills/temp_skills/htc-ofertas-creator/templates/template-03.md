# Template 03: Oferta de Serviço Done-For-You (DFY)

## Estrutura para Ofertas de Serviço "Pronto" (Consultoria Implementacional)

Use este template para criar ofertas onde você ou sua equipe IMPLEMENTAM a solução para o cliente, não apenas ensinam como fazer.

---

## Diferença Chave: Coaching vs DFY

**Coaching/Mentoria:**
- Ensina COMO fazer
- Cliente implementa
- Preço menor (R$3k-10k)
- Resultado depende do cliente

**Done-For-You (DFY):**
- Você FAZ pelo cliente
- Você implementa
- Preço maior (R$10k-100k+)
- Resultado garantido (ou sua reputação sofre)

**Quando usar DFY:**
- Cliente tem dinheiro mas não tempo
- Implementação é complexa/técnica
- Cliente já tentou e falhou sozinho
- Você tem equipe/processo escalável

## Checklist de Qualidade para DFY

Antes de lançar sua oferta DFY:

**Clareza do Escopo:**
- [ ] O que você faz está 100% claro
- [ ] O que cliente faz está 100% claro (mínimo)
- [ ] Entregáveis são tangíveis e mensuráveis
- [ ] Prazo realista (nem muito curto nem muito longo)

**Valor Percebido:**
- [ ] Cada entregável tem valor estimado
- [ ] Total é 2-3x o preço
- [ ] Diferença clara entre "você faz" vs "cliente faz"

**Garantia Inteligente:**
- [ ] Protege o cliente (se não funcionar)
- [ ] Protege você (condições claras)
- [ ] Baseada em entregável, não "satisfação"

**Limitação:**
- [ ] Capacidade real definida (não inventada)
- [ ] Razão genuína para limitação
- [ ] Vagas restantes visíveis

**Processo de Venda:**
- [ ] Call de qualificação (não venda direta)
- [ ] Proposta personalizada após call
- [ ] Ambos podem dizer "não é fit"

## Erros Comuns em DFY

❌ **Não faça:**
- Escopo vago ("nós ajudamos você")
- Entregáveis indefinidos
- Prazo irreal (prometer muito rápido)
- Garantia de "satisfação" (subjetivo)
- Sem limitação de capacidade
- Vender sem qualificar

✅ **Faça:**
- Escopo cristalino ("nós entregamos X, Y, Z")
- Entregáveis mensuráveis e tangíveis
- Prazo realista com buffer
- Garantia baseada em entregável
- Limitado por capacidade real
- Qualificar antes de vender

## Quando Usar DFY vs Coaching

**Use DFY quando:**
- Cliente tem dinheiro mas não tempo
- Implementação é complexa/técnica
- Cliente já falhou sozinho antes
- Você tem equipe/processo escalável
- Margem permite (mínimo 50%)

**Use Coaching quando:**
- Cliente tem tempo e orçamento limitado
- Implementação é simples
- Cliente quer aprender o processo
- Você é solopreneur ou equipe pequena
- Escala através de volume

**Hybrid (melhor dos dois):**
- Programa com coaching + módulos DFY
- Ex: "Nós configuramos X, você gerencia Y"
- Preço intermediário
- Melhor taxa de sucesso
