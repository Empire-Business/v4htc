# Módulo 1: O Que Você Vai Construir

---

## Objetivo deste Módulo

Entender que tipos de aplicativos você pode construir para o seu negócio e escolher o **seu primeiro projeto**.

---

## O Que Você Pode Construir

Com Claude Code ou Lovable, você pode criar:

### 1. Landing Pages
- Página de vendas do seu produto
- Página de captura de leads
- Página de lançamento
- Página para evento

**Exemplo:** Uma página para vender seu consultoria com formulário de contato e agendamento.

### 2. Dashboards
- Painel de métricas do negócio
- Dashboard de vendas
- Dashboard de marketing
- Dashboard financeiro

**Exemplo:** Ver em tempo real quanto vendeu hoje, quais produtos mais vendem, ticket médio.

### 3. CRMs Simples
- Gestão de clientes
- Pipeline de vendas
- Histórico de interações
- Lembretes de follow-up

**Exemplo:** Ver todos os seus prospects, em que etapa estão, quando foi o último contato.

### 4. Sistemas de Agendamento
- Calendário online
- Agendamento de serviços
- Confirmação automática
- Bloqueio de horários

**Exemplo:** Clientes agendarem horários na sua clínica sem precisar te ligar.

### 5. Áreas de Cliente
- Portal do cliente
- Histórico de pedidos
- Downloads de produtos
- Suporte

**Exemplo:** Seus clientes acessam um site para ver pedidos anteriores e baixar notas fiscais.

### 6. Aplicativos Internos
- Gestão de estoque
- Controle de tarefas
- Sistema de financeiro
- Fluxo de caixa

**Exemplo:** Sua equipe registra vendas e você vê tudo em tempo real.

### 7. Marketplaces
- Plataforma com vários vendedores
- Catálogo de produtos
- Sistema de avaliações

**Exemplo:** Um site onde vários fornecedores vendem para seus clientes.

### 8. Portfólios
- Showcase de produtos
- Galeria de trabalhos
- Cases de sucesso

**Exemplo:** Mostre seus casos de sucesso para prospects interessados.

---

## Comece com Algo Simples

**ERRO COMUM:** Tentar construir algo complexo no primeiro projeto.

**RECOMENDAÇÃO:** Comece com um sistema que resolva **UMA dor específica**.

### Exemplos de Primeiros Projetos

| Negócio | Primeiro Projeto | Por Que Começar com Ele |
|---------|------------------|------------------------|
| Clínica | Agenda simples | Dor urgente: horários caos |
| E-commerce | Dashboard de vendas | Dor: não sabe quanto vendeu |
| Consultoria | Landing page + formulário | Dor: não capta leads |
| Restaurante | Sistema de delivery | Dor: pedidos no WhatsApp |
| Imobiliária | Gestão de leads | Dor: perde clientes |

---

## Checklist: Seu Primeiro Projeto

Antes de escolher, pergunte-se:

- [ ] **Qual é a DOR mais urgente do meu negócio hoje?**
- [ ] **Se eu tivesse esse sistema funcionando amanhã, o que mudaria?**
- [ ] **Quem vai usar este sistema?** (só eu? minha equipe? clientes?)
- [ ] **Que informações o sistema precisa guardar?**
- [ ] **Como eu sei que o sistema está funcionando?**

---

## Exemplos Reais

### Caso 1: Maria - Salão de Beleza

**Dor:** Clientes ligam o dia todo para agendar. Maria esquece de anotar. Horários se sobrepõem.

**Primeiro Projeto:** Sistema de Agendamento
- Cliente vê horários disponíveis
- Escolhe o serviço
- Agenda sozinha
- Maria recebe notificação

**Tempo para construir:** 2-3 dias
**Economia de tempo:** 2 horas por dia

### Caso 2: João - E-commerce de Moda

**Dor:** Não sabe quanto vendeu, o que está vendendo mais, quanto tem de estoque.

**Primeiro Projeto:** Dashboard de Vendas
- Vendas do dia
- Produtos mais vendidos
- Estoque baixo (alerta)
- Ticket médio

**Tempo para construir:** 1-2 dias
**Benefício:** Decisões baseadas em dados

### Caso 3: Ana - Consultoria de Marketing

**Dor:** Perde prospects porque não tem follow-up organizado.

**Primeiro Projeto:** CRM Simples
- Cadastro de leads
- Pipeline (novo, contatando, negociando, fechado)
- Lembretes de follow-up
- Histórico de conversas

**Tempo para construir:** 2-3 dias
**Benefício:** Aumentou fechamento em 40%

## Framework de Escolha: IMPACTO x ESFORÇO

```
        ALTO IMPACTO
             |
     PRIORIDADE    FAZER DEPOIS
             |
             +---------------
             |               |
   BAIXO ESFORÇO        ALTO ESFORÇO
```

**COMECE POR:** ALTO IMPACTO + BAIXO ESFORÇO

### Exemplos por Categoria:

**FAZER AGORA (Alto Impacto, Baixo Esforço):**
- Landing page com formulário
- Dashboard de métricas principal
- Agenda simples

**PRIORIDADE (Alto Impacto, Alto Esforço):**
- Sistema completo de agendamento
- CRM com automação
- Marketplace

**FAZER DEPOIS (Baixo Impacto, Baixo Esforço):**
- Página institucional simples
- Blog básico

**EVITAR (Baixo Impacto, Alto Esforço):**
- Sistema super complexo que ninguém vai usar
- App mobile quando um site resolveria
