# Guidelines de Uso da Marca HTC
## High Ticket Code - Brand Guidelines

**Versão:** 1.0
**Data:** Fevereiro 2026

## Como Usar a Marca HTC

### 1. Logotipo

#### Versão Principal

O logotipo HTC deve ser usado sempre que possível em sua versão completa.

**Especificações:**
- Formato: Horizontal (padrão)
- Cor: Primary Blue (#2563EB) ou Primary Dark (#1E3A8A)
- Fundo: Preferencialmente claro ou com contraste adequado
- Espaçamento mínimo: Igual à altura do logotipo

#### Versões Alternativas

**Stacked (Vertical):**
- Usar quando espaço horizontal é limitado
- Mantém proporções e espaçamentos

**Monocromática:**
- Usar quando cores não são possíveis
- Mantém forma e legibilidade

**Reversed (Branca):**
- Usar sobre fundos escuros ou coloridos
- Garantir contraste suficiente

### 2. Área de Proteção

**Espaçamento Mínimo:**
- Manter espaço livre ao redor do logotipo
- Distância mínima: 50% da altura do logotipo
- Nunca sobrepor texto ou outros elementos

**Exemplo:**
```
┌─────────────────────────────────┐
│                                 │ ← 50% altura (mínimo)
│   ┌─────────────────────┐       │
│   │      HTC LOGO       │       │
│   └─────────────────────┘       │
│                                 │ ← 50% altura (mínimo)
└─────────────────────────────────┘
```

### 3. Tamanho Mínimo

**Digital:**
- Largura mínima: 120px
- Altura mínima: 30px (horizontal)

**Impresso:**
- Largura mínima: 25mm
- Altura mínima: 6mm (horizontal)

**Exceção:** Favicons, avatares, onde a versão simplificada pode ser usada

## O Que FAZER

### 1. Aplicação de Cores

**DO:**
- Usar a paleta de cores oficial
- Aplicar Glassmorphism em interfaces digitais
- Manter contraste adequado para acessibilidade
- Usar gradientes sutis e elegantes
- Testar em diferentes contextos (light/dark mode)

### 2. Tipografia

**DO:**
- Usar Inter como fonte principal
- Seguir hierarquia tipográfica estabelecida
- Manter legibilidade em todos os tamanhos
- Usar JetBrains Mono para código
- Respeitar line-heights e letter-spacings

### 3. Layout e Composição

**DO:**
- Usar grid system consistente
- Manter espaçamentos uniformes
- Aplicar border-radius do design system
- Usar sombras sutis para depth
- Manter alinhamentos precisos

### 4. Imagens e Ilustrações

**DO:**
- Usar imagens de alta qualidade
- Estilo: clean, moderno, minimalista
- Cores: complementares à paleta HTC
- Contexto: tecnologia, negócios, inovação
- Evitar clichês (stock photos óbvias)

### 5. Tom de Voz

**DO:**
- Ser autêntico e genuíno
- Usar linguagem clara e acessível
- Demonstrar empatia com o empresário
- Ser orientado à ação
- Manter consistência em todos os canais

### 6. Glassmorphism

**DO:**
- Aplicar blur (backdrop-filter) em elementos glass
- Usar bordas translúcidas sutis
- Manter transparência apropriada
- Testar em diferentes backgrounds
- Considerar performance

### 7. Acessibilidade

**DO:**
- Garantir contraste WCAG AA (mínimo)
- Fornecer alt text para imagens
- Usar semantic HTML
- Testar com leitores de tela
- Considerar daltonismo

## O Que NÃO FAZER

### 1. Logotipo

**DON'T:**
- Não distorcer o logotipo (esticar, comprimir)
- Não alterar as cores
- Não adicionar efeitos (sombra, gradiente, outline)
- Não usar versões de baixa resolução
- Não modificar a tipografia do logotipo
- Não sobrepor em backgrounds complexos
- Não usar o logotipo muito pequeno

**Exemplos de uso incorreto:**
```
❌ HTC (estilizado, distorcido)
❌ htc (minúsculas)
❌ H.T.C. (pontos)
❌ HTC com cores diferentes
❌ HTC com efeitos 3D
```

### 2. Cores

**DON'T:**
- Não usar cores fora da paleta oficial
- Não usar cores semânticas incorretamente (ex: error para sucesso)
- Não ignorar contraste em nome do estilo
- Não usar gradientes muito fortes
- Não misturar muitas cores

**Exemplos de combinações a evitar:**
```
❌ Primary + Error (confusão visual)
❌ Texto claro sobre fundo claro
❌ Gradiente com 5+ cores
❌ Neon ou cores muito saturadas
```

### 3. Tipografia

**DON'T:**
- Não usar fontes fora do design system
- Não usar mais de 2-3 famílias de fonte
- Não usar tamanhos fora da escala
- Não usar weights extremes (100, 900)
- Não usar letter-spacing excessivo

**Exemplos a evitar:**
```
❌ Comic Sans, Papyrus, fontes decorativas
❌ Font sizes: 11px, 73px, 127px (fora da escala)
❌ Texto em ALL CAPS longo
❌ Letter-spacing: 0.5em ilegível
```

### 4. Layout e Composição

**DON'T:**
- Não ignorar grid e alinhamento
- Não usar espaçamentos inconsistentes
- Não amontoar elementos
- Não deixar muito espaço em branco sem propósito
- Não quebrar regras de proximidade

### 5. Glassmorphism

**DON'T:**
- Não usar transparência excessiva (ilegível)
- Não esquecer bordas (elementos somem no fundo)
- Não usar em backgrounds muito movimentados
- Não ignorar performance (backdrop-filter é pesado)
- Não misturar muitos estilos diferentes

**Exemplos a evitar:**
```
❌ background: rgba(255, 255, 255, 0.1) (muito transparente)
❌ Glass sem border (vira nothing)
❌ Glass sobre glass sobre glass (confuso)
❌ Backdrop-filter em elementos animados (performance)
```

### 6. Conteúdo

**DON'T:**
- Não prometer resultados irreais
- Não usar jargão técnico sem explicação
- Não ser condescendente ou arrogante
- Não copiar concorrentes
- Não usar hype excessivo

### 7. Acessibilidade

**DON'T:**
- Não ignorar contraste de cores
- Não depender apenas de cor para comunicar informação
- Não usar texto muito pequeno
- Não esquecer de alt text
- Não fazer_dependente_de_cor_somente

## Exemplos de Aplicação

### 1. Website

**CORRETO:**
- Hero com Primary gradient background
- Navigation com glass effect
- Cards com glass-card styling
- CTAs com Primary 600
- Texto com hierarquia clara
- Contraste WCAG AA ou superior

**INCORRETO:**
- Hero com 7 cores diferentes
- Navigation sem blur (glass incompleto)
- Cards sem borda (somem no fundo)
- CTAs com cores aleatórias
- Texto todo mesmo tamanho
- Contraste insuficiente

### 2. Apresentação (Slides)

**CORRETO:**
- Título com Plus Jakarta Sans, 6xl
- Subtítulo com Inter, 2xl
- Body com Inter, base
- Código com JetBrains Mono, sm
- Glass cards para conteúdo
- Logo no canto inferior direito

**INCORRETO:**
- Título com Comic Sans
- Texto em 11px
- Código em fonte regular
- Cards sem estilo
- Logo gigante no centro

### 3. Email Template

**CORRETO:**
- Logo no topo
- Headline com 2xl
- Body com base
- CTA destacado
- Assinatura com sm
- Footer com xs

**INCORRETO:**
- Sem logo
- Texto todo mesmo tamanho
- CTA confuso
- Sem assinatura
- Sem footer

### 4. Social Media

**CORRETO:**
- Logo compacto
- Headline curta e impactante
- Visual limpo
- CTA claro
- Hashtags relevantes

**INCORRETO:**
- Logo distorcido para caber
- Texto infinito
- Visual poluído
- Sem CTA
- 20 hashtags aleatórias

## Contextos Específicos

### 1. Co-branding

**Quando usar HTC + outro logo:**
- Manter proporções adequadas
- Separar com espaço claro
- Usar divisor sutil quando necessário
- Ambos logos devem ter peso visual similar

**Exemplo:**
```
┌─────────────────────────────────────────────────┐
│                                                 │
│  [HTC LOGO]        |        [PARCEIRO LOGO]    │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 2. Fundos Escuros

**RECOMENDAÇÕES:**
- Usar logo reversed (branco)
- Aumentar contraste de texto
- Usar glass-dark em vez de glass-light
- Testar legibilidade

**EVITAR:**
- Logo principal azul sobre escuro (contraste baixo)
- Texto escuro sobre fundo escuro
- Efeitos glass muito sutis

### 3. Fundos Coloridos

**RECOMENDAÇÕES:**
- Usar logo com contraste adequado
- Manter simplicidade
- Evitar muito ruído visual

**EVITAR:**
- Logo perdido no fundo
- Muitas cores competindo
- Falta de hierarquia visual

### 4. Material Impresso

**RECOMENDAÇÕES:**
- Usar CMYK para impressão
- Verificar resolução mínima (300dpi)
- Testar cores em prova
- Considerar papel (afeta cores)

**EVITAR:**
- RGB para impressão (cores ficam diferentes)
- Resolução baixa
- Ignorar configuração de impressora

## Erros Comuns

### 1. Contraste Insuficiente

**Problema:** Texto claro sobre fundo claro

**Solução:** Usar darker text ou lighter background

### 2. Glass Sem Bordas

**Problema:** Elementos glass somem no fundo

**Solução:** Sempre adicionar border (1px, rgba(255,255,255,0.18))

### 3. Muitos Efeitos

**Problema:** Glass + gradient + shadow + reflection = caos

**Solução:** Menos é mais. Escolha 1-2 efeitos

### 4. Inconsistência

**Problema:** Cada página/peça parece de marca diferente

**Solução:** Criar checklist e seguir rigorosamente

### 5. Ignorar Performance

**Problema:** Backdrop-filter em tudo = site lento

**Solução:** Usar com moderação, especialmente em mobile

## Checklist de Uso

Antes de publicar qualquer material, verifique:

### Logotipo
- [ ] Versão correta usada?
- [ ] Tamanho adequado?
- [ ] Espaçamento suficiente?
- [ ] Sem distorções?
- [ ] Cores corretas?

### Cores
- [ ] Paleta oficial usada?
- [ ] Contraste WCAG AA?
- [ ] Cores semânticas corretas?
- [ ] Consistente com brand?

### Tipografia
- [ ] Fonte correta usada?
- [ ] Hierarquia respeitada?
- [ ] Tamanhos na escala?
- [ ] Legível?

### Layout
- [ ] Grid/alinhações ok?
- [ ] Espaçamentos consistentes?
- [ ] Comp. balanceada?
- [ ] Responsive (se digital)?

### Conteúdo
- [ ] Tom de voz correto?
- [ ] Sem erros gramaticais?
- [ ] CTAs claros?
- [ ] Autêntico?

### Acessibilidade
- [ ] Contraste suficiente?
- [ ] Alt text em imagens?
- [ ] Semantic HTML (se web)?
- [ ] Testado com leitor?

### Glassmorphism
- [ ] Backdrop-filter aplicado?
- [ ] Bordas presentes?
- [ ] Transparência adequada?
- [ ] Funciona no background?

## Processo de Aprovação

### 1. Auto-revisão

Use o checklist acima antes de compartilhar

### 2. Peer Review

Peça para alguém da equipe revisar

### 3. Design Lead (se crítico)

Materiais principais devem ser aprovados pelo Design Lead

### 4. Teste

Teste em contexto real (web, mobile, impresso)
